HTTP Method란?

 

웹 개발에서 HTTP 메서드는 클라이언트가 서버에 요청하는 동작을 나타냅니다. 이 메서드는 리소스에 대한 작업을 명확하게 정의하여 서버가 요청을 처리하는 데 도움을 줍니다. 이 블로그 게시글에서는 널리 사용되는 HTTP 메서드 5가지, GET, POST, PATCH, PUT, DELETE에 대해 자세히 알아봅니다.

 

HTTP Method의 종류

  • GET

    리소스의 조회를 요청하는 메서드

  • POST

    리소스의 생성을 요청하는 메서드

  • PUT

    리소스의 수정을 요청하는 메서드

  • PATCH

    리소스의 수정을 요청하는 메서드

  • DELETE

    리소스의 삭제를 요청하는 메서드

 

HTTP Method를 사용하는 이유

  1. 1. 리소스와 동작을 분리하기 위해서

    HTTP 프로토콜은 URL을 통해 리소스를 식별합니다. 하지만 URL만으로는 클라이언트가 서버에 어떤 작업을 요청하는지 알 수 없습니다. 예를 들어, 특정 게시글을 조회하는 것인지, 수정하는 것인지, 삭제하는 것인지를 구분할 수 없습니다. 따라서 HTTP 메서드를 사용하여 클라이언트가 서버에 요청하는 동작을 명시적으로 표현할 수 있습니다.

    2.  

  2. 2. 요청의 의미를 명확하게 하기 위해서

    3. HTTP 메서드는 각각 고유한 의미를 가지고 있습니다. 예를 들어, GET 메서드는 리소스의 조회를, POST 메서드는 리소스의 생성을 의미합니다. 따라서 HTTP 메서드를 사용하면 클라이언트의 의도를 명확하게 표현할 수 있습니다. 이렇게 하면 서버는 요청을 정확하게 처리하고 오류를 줄일 수 있습니다.

 

GET

 

사용목적: 리소스 조회

 

GET 메서드는 서버에 특정 리소스를 요청하는 데 사용됩니다. 웹 브라우저가 웹 페이지를 요청할 때, API가 데이터를 가져올 때 등 다양한 상황에서 사용됩니다.

서버에 전달하고 싶은 데이터는 쿼리스트링을 통해서 잔달합니다.

 

예시

 

클라이언트에서 10번 게시글을 가져오고 싶을 때 /post/10 으로 요청

클라이언트가 GET으로 요청하면 서버는 해당 데이터를 응답해준다.

 

 

POST

 

사용목적: 리소스 조회

 

POST 메서드는 서버에 새로운 리소스를 생성하는 데 사용됩니다. 웹 폼을 통해 데이터를 전송하거나, 새로운 사용자를 등록하는 등 다양한 상황에서 사용됩니다. 보통은 HTTP Body에 데이터를 담아서 서버에 데이터를 전송합니다.

 

예시

 

클라이언트에서 title: 블로그 제목... content: 블로그 내용... 이라는 데이터 생성을 서버에 요청

서버는 해당 데이터를 받아서 DB에 저장한다.

 

 

PUT

 

사용목적:리소스 전체 수정

 

PUT 메서드는 리소스를 전체적으로 대체하는 데 사용됩니다. 기존 리소스가 존재하면 새 리소스로 대체되고, 리소스가 존재하지 않으면 새 리소스가 생성됩니다.

 

예시

 

수정하려는 데이터가 이미 존재할 경우 해당 데이터를 덮어쓴다.

10번 게시글이 이미 존재하기 때문에 서버에 저장되어 있는 데이터를 클라이언트가 보낸 데이터로 덮어쓴다.

 

수정하려는 데이터가 존재하지 않을 경우 데이터를 새로 생성한다.

서버에 10번이라는 게시글이 없기 때문에 클라이언트가 보낸 데이터를 새로 생성한다.

 

 

PATCH

 

 

사용목적:리소스 부분 수정

 

PATCH 메서드는 리소스의 일부 속성만 변경하는 데 사용됩니다. 전체 리소스를 대체하는 PUT 메서드와 달리, PATCH 메서드는 변경하려는 속성만 지정하여 전송합니다.

 

예시

 

블로그의 제목만 수정하려는 경우

 

 

DELETE

 

 

사용목적:리소스 삭제

 

DELETE 메서드는 서버에서 리소스를 삭제하는 데 사용됩니다. 리소스가 삭제되면 더 이상 사용할 수 없게 됩니다.

 

예시

 

10번 게시글을 삭제합니다.
